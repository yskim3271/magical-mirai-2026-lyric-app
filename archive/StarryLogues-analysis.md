## 컨셉 구조

곡 = **은하**, 가사 = **별**. 2025 테마 "星河一天(성하일천)"에 맞춰:
- 가사(창작에 담긴 마음)끼리 별자리처럼 선으로 이어짐
- 칠석 **단자쿠(短冊, 소원 쪽지)** = 타인의 마음. 시야를 움직여 단자쿠를 찾아 모으면 가사에 닿아 빛나며 사라짐
- 재생 = "관측 시작", 곡 종료 = "관측 완료 + 모은 단자쿠 수 집계"라는 **플라네타륨 관측 프레이밍**
- 시간·장소를 바꿔도(별자리 모양이 변할 만큼 시간이 흘러도) 메인 콘텐츠는 실행됨 — "한번 표현된 마음은 계속 남는다"는 메시지를 기능으로 구현

## 구현된 기능 인벤토리

### 1. 가사 → 별 변환 (TextAlive 데이터 총동원)
- 단어 단위로 별이 생성되어 천구(반경 40)에 배치, canvas 텍스트 스프라이트로 렌더
- **감정·음량 데이터를 별의 물리 속성에 매핑** (main.js:3569-3591):
  - arousal(각성도) → **B-V 색지수** → 흑체 온도 → RGB (실제 항성 색 물리 공식: bv2Temperature → temperatureToXYZ → xyzToRgb)
  - valence(긍정도) → 밝기 (0.5~2.0)
  - vocalAmplitude(보컬 음량) → 별 크기 (0.7~1.4)
- **onTimerReady에서 전곡 단어를 사전 스캔해 V/A·음량의 min/max를 구해 정규화** (main.js:3488-3499) — 곡마다 데이터 범위가 달라도 연출 일관성 유지. 그대로 가져다 쓸 패턴.

### 2. 품사 기반 단어 병합 (가독성 핵심 기술)
main.js:3600-3700. TextAlive 단어에 붙는 품사(pos)·언어(language) 정보를 사용:
- 조사(pos="S")를 앞 명사에 붙임 — front/middle/back 조사 리스트로 분기
- 명사+명사 연결은 **두 단어의 비트 간격이 1마디 이내일 때만** 병합 (findBeat index 비교)
- 영어는 공백으로 join, 일본어는 그대로 연결
- 결과: "歌詞" 같은 토막이 아니라 "歌詞を" 단위의 자연스러운 덩어리로 별이 뜸

### 3. 실제 천문 시뮬레이션 (배경이지만 차별화 핵심)
- **히파르코스 성표**(등급 ≤6.5)에서 적경·적위·B-V·고유운동 로드 (빌드 타임에 Python 스크립트로 JSON 생성 — `Make_HIP2StarNamejson.py` 등)
- 고유운동 + 세차운동(astronomia) 보정 → 율리우스일/GST/LST 계산 → 관측지 기준 고도·방위각 변환 (README에 수식 전체 해설)
- **Geolocation API로 사용자 실제 위치** 반영, 관측 시각·장소 수동 변경 가능
- 별자리 선·명칭은 Stellarium 공개 데이터(일부 개변) + 국립천문대 일본어 대응표
- 기능적 역할도 부여: 시점 회전량을 인지시키는 시각 기준점 (90도 제한 시 멈췄는지 판단)

### 4. 인터랙션
- 시점: 드래그 + **자이로 센서**(DeviceOrientationControls, iOS는 requestPermission 처리 — main.js:5194)
- Raycaster로 단자쿠 픽킹 (3곳에서 setFromCamera/intersectObjects)
- 위시리스트: 하드코딩된 소원 텍스트 풀에서 매 플레이 **랜덤 30개** 선택 (main.js:4302~, "수험 합격", "좋아하는 사람에게 마음이 닿기를" 등 — AI 생성 금지 규정 때문에 전부 수작업 텍스트)
- **단자쿠 27개 초과 수집 시 가사 등장 파티클이 불꽃놀이 모드로 전환** — 히든 보상 설계

### 5. 파티클 수학 (README에 해설 포함)
- ParticleExplosion 클래스 (main.js:4055): 가사 위치의 **접평면 위에서** 파티클이 퍼지도록 법선 벡터와 직교하는 축 2개를 외적으로 구해 cos/sin 합성 — 화면 어디서든 자연스러운 방향으로 퍼짐

### 6. MV 썸네일 모핑 (규정 예외의 영리한 활용)
- prepareMVimg (main.js:5020): **별 파티클들이 곡 MV 썸네일 이미지 모양으로 변형** — 썸네일을 캔버스로 샘플링해 픽셀 위치를 파티클 목표 좌표로. 공식 에셋 중 유일하게 허용되는 "MV 썸네일"을 정확히 짚어서 사용.

### 7. 결과물 내보내기
- **CubeCamera로 6방향 캡처 → 등장방형(파노라마) 합성 → PNG 다운로드** (get_6dir_img, directionToCubeUV, toDataURL) — "내가 만든 별하늘"을 가져가게 함. 일반 투표에서 공유 유발 장치.

### 8. 제품화 디테일
- **지정곡 6곡 전부 선택 가능** (load_music: createFromSongUrl 6개 분기, 곡별 고정 ID) — 은하 선택 UI
- 첫 실행 설명 오버레이, 로딩 화면, 설정 패널(색상 모드, 별자리 명칭 토글, 선 색), 망원경 진입 연출, 진행 게이지
- `app.managed` 체크로 TextAlive 호스트 연결/단독 실행 분기 (onAppReady)
- 곡 종료 판정: `player.timer.position >= duration - 100` → "관측 완료" 화면 전환
- 폰트 4종 번들, README에 동작 확인 환경 명시 (Win Chrome / Android Chrome / iOS Safari + Safari 메모리 경고)

## 왜 이겼는가 — 우리가 가져갈 교훈

1. **"웹이어야만 하는 이유"가 겹겹이**: 시점 조작, 자이로, 내 위치의 진짜 밤하늘, 내가 모은 단자쿠, 파노라마 저장 — 영상으로 절대 대체 불가능한 요소가 5개 이상.
2. **TextAlive 데이터 채널을 전부 사용**: 단어 타이밍, 품사, 언어, 비트, V/A 감정값, 보컬 음량, MV 썸네일까지. 심사위원(API 개발진 포함)에게 "API를 깊게 이해했다"는 신호.
3. **테마 정합성**: 행사 테마(星河一天) × 칠석 × "창작은 사람을 잇는다"는 매지컬 미라이 정신 — 심사 기준 '혁신성(미래 창작 문화 예감)'을 정면 공략.
4. **빌드 타임 데이터 파이프라인**: 정적 앱 제약을 Python 전처리 + JSON 번들로 우회. 우리도 6곡 공통 메타데이터나 보조 분석값을 JSON으로 굽는 같은 패턴 사용 가능.
5. **코드 구조는 의외로 투박**: 5,262줄 단일 파일, 주석 처리된 디버그 코드 다수, 오타 변수명(rylic) — **심사는 체험 품질을 보지 코드 미학을 보지 않는다.** 시간 배분의 교훈.
6. 개인 제작 + "그림 실력 없음"을 컨셉으로 흡수(졸라맨 단자쿠) — 에셋 부족을 약점이 아니라 유머로.
