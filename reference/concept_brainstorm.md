# 컨셉 브레인스토밍 — 전곡 대응

전제: 대상곡은 Magical Mirai 2026 악곡 콘테스트 수상작 6곡 전체다.
특정 곡의 가사 해석에 고정하지 않고, **TextAlive 데이터만으로 6곡 모두에서 성립하는 메커니즘**을 먼저 설계한다.

## 구현 전제

1차 목표는 다음이다.

- 6곡 선택 UI
- 곡별 버전 고정 URL/ID 관리
- TextAlive 데이터 로더 공통화
- 단어/문자/비트/코러스/음량/V-A 데이터 기반 연출
- 곡이 바뀌어도 같은 규칙으로 성립하는 시각 시스템

## 곡 비의존 데이터 도구상자

곡이 바뀌어도 쓸 수 있는 입력 신호:

- 단어/문자 타이밍 + 품사·언어 → 표시 단위 병합과 가독성
- 비트/BPM → 펄스, 반복 리듬, 화면 전환 타이밍
- 사비 세그먼트(`findChorus`) → 클라이맥스 전환
- 단어별 valence/arousal + 곡 중앙값 → 색, 밀도, 움직임 성격
- 보컬 음량(`getVocalAmplitude`) → 크기, 밝기, 높이, 힘
- 곡 통계 → 곡별 성격 프리셋. 빠른 곡은 높은 밀도와 강한 움직임, 낮은 valence는 어두운 색/흐림, 높은 valence는 밝은 색/개방감.

## 후보 컨셉

### A. 리릭 분수 쇼

가사가 소리의 물기둥처럼 솟고, 비트와 보컬 음량에 맞춰 분수/입자/빛이 반응한다.

- 장점: 음악, 호수, `ソナーレ` 테마와 잘 맞음
- 보완점: 관람형에 머물지 않도록 탭/드래그 반응이나 결과물 저장이 필요

### B. 파문 간섭 캔버스

모든 발성이 수면의 파원이 되고, 곡이 진행되며 곡마다 다른 파문 무늬를 만든다.

- 장점: 6곡별 차이가 자연스럽게 생김
- 보완점: 추상성이 강하므로 가사 가독성 레이어를 별도로 설계해야 함

### C. 사운드 스코어 / 악보 도시

가사와 비트를 악보, 궤도, 도시 불빛처럼 배치한다. 사용자는 곡을 선택해 각 곡의 리듬 구조를 탐색한다.

- 장점: 전곡 대응 UI와 잘 맞음
- 보완점: Magical Mirai 2026의 호수/음악 테마를 시각 언어로 충분히 끌어와야 함

### D. 거울/반사 세계

상단에는 현재 가사, 하단에는 반사/잔상/메아리를 배치한다. arousal이 높을수록 반사가 크게 흔들린다.

- 장점: 구현 효율이 높고 6곡 모두에 적용 가능
- 보완점: 단독 메인 컨셉보다는 보조 연출에 가깝다

## 유지할 판단 기준

- 특정 가사 블록 분석보다 TextAlive 데이터 채널을 우선한다.
- 6곡 모두에서 30초 안에 차이가 보이게 한다.
- 스마트폰에서 버벅이면 버린다.
- 저장/공유/상호작용 중 하나는 반드시 넣어 "영상으로 대체 불가능한 이유"를 만든다.
- 최종 README에는 지원 곡, 동작 환경, 조작법, 라이선스 표기를 명확히 쓴다.
